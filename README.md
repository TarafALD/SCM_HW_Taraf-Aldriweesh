This repository is created for the SWE 442 Software Configuration Management course. It demonstrates understanding of version control, branching strategies, and issue tracking using GitHub
Software Configuration Management Best Practices Effective SCM involves practices like using small, frequent commits, adopting a standard branching strategy (e.g., Gitflow or GitHub Flow), writing descriptive commit messages, and diligently linking all code changes to a corresponding issue or task for traceability
